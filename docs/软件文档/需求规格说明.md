# 1.2 系统概述

测试性建模与交互式故障诊断一体化集成开发环境软件可实现软件的系统集成、基本器件库、诊断、单点故障定位功能。系统集成功能可以设置各类设备建模与集成建模，如机械设备、电气设备、液压器件等。基本器件库功能可用对电气器件进行查找、编辑、删除、复制，以及对基本器件库的查看、非标准器件模型建模与校验。诊断功能用于进行交互式诊断并生成交互式诊断方案。单点故障定位功能用于对液压泵站启动功能故障进行定位。

# 3需求

# 3.1 要求的状态和方式

完成测试性建模与交互式故障诊断一体化集成开发环境软件开发及相关配套文档（包括开发总结报告），实现对建模、测试性评价、电气、电子、液压、机械、基本器件库管理、交互式诊断方案、单点故障定位等。本系统所用的需求分析方式为结构化分析法。完成测试性建模与交互式故障诊断一体化集成开发环境软件及相关配套文档（包括需求规格说明、设计报告、使用手册、测试报告）。

# 3.2 功能描述

测试性建模与交互式故障诊断一体化集成开发环境软件主要功能包括建模、测试性评价、电气、电子、液压、机械、基本器件库管理、交互式诊断方案、单点故障定位等，具体功能如下

1）建模与测试性评价功能。提供可视化的多领域建模操作环境，支持用户完成机械、电气、液压器件设备的独立建模，以及机械、电气、液压设备的跨领域集成建模，快速构建系统完整拓扑结构；基于已搭建的机械、电气、液压集成模型，提供标准化的测试性评价操作入口，支持自动调用底层分析引擎，完成集成模型的测试性指标分析与结果呈现，为后续诊断工作提供数据支撑。  
2）电气与基本器件库管理功能。针对电气器件，提供图形化的操作界面，支持用户对电气器件进行查看、查找、编辑、删除、复制等全流程管理操作；针对基本器件库，提供直观的图形化浏览入口，方便用户查询器件核心参数与模型信息；同时支持非标准器件模型的自定义建模，用户可自主设置器件属性、功能逻辑与端口定义，并提供非标准器件模型校验工具，确保模型的合规性与适配性。  
3）交互式诊断方案生成功能。提供可视化的交互式诊断方案操作界面，支持用户通过菜单、按钮等便捷操作触发诊断流程，系统将自动适配已构建的系统模型，生成针对性的交互式诊断方案，为故障排查提供清晰指引。  
4）单点故障定位功能。针对液压泵站启动功能故障等具体应用

场景，提供引导式的单点故障定位操作环境，支持接收用户输入的测试数据或操作指令，通过底层推理引擎进行分析运算后，将故障定位结果（如故障部件、关联测试点、排查建议）以高亮标记或列表形式呈现，助力用户精准完成单点故障定位。

# 3.9 软件质量因素

a) 功能性要求

系统满足研制总要求或需求分析报告提出的功能和性能要求；

b) 可靠性要求

针对各种不同的数据、不同的操作、正确和错误的输入条件，都应能进行正确处理。在系统环境出错、用户操作失误、数据本身错误或违反指定接口的情况下，应尽可能提供友好的提示；

c) 易用性要求

系统提供友好的人机界面，为便于理解对用户输入尽可能提供详尽的说明文字，在适当的时机显示提示文字，提供完备的用户操作手册；

d) 维护性要求

系统应采用便于维护的构件化、模块化设计，提供必要的调试和测试工具，提供维护所必须的软件文档。

# 3.10 设计和实现约束

本系统应考虑如下的设计约束：

a) 软件结构采用 C/S 模式，数据的存储采用 Access 等数据库管理系统，使用 C# 编程语言；  
b) 统一数据结构及数据称谓。

# 3.11 人员需求

a)需对系统使用人员进行必要的培训，使参训人员熟悉本系统的工作原理、操作使用、维护管理，能够熟练掌握本系统的使用、维护与管理，以保证系统交付后能够正常使用；  
b) 培训内容包括对相关专业技术知识的培训、对应用软件的操作使用培训、对系统管理和系统维护的培训。培训时需准备相应的培训教材；  
c) 所有的使用人员都应有计算机操作的基本知识，熟悉 Microsoft Windows 风格的软件界面。

# 3.12 培训需求

需对使用系统的系统管理员和普通用户两种角色进行必要的培训，使系统管理员对系统提供的系统维护、用户管理等功能能够熟练操作。

培训内容包括对应用软件的操作使用培训、对系统管理和系统维护的培训。培训时需准备相应的培训教材。

# 3.13 软件保障需求

a) 以系统版本、系统补丁的方式提供系统升级维护，提供一键式的系统升级安装和补丁更新方式，便于用户操作。  
b)系统正常使用情况下由用户进行维护，如遇重大问题由研发人员到现场维护。
